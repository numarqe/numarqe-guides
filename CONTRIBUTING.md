# Contributing to Numarqe Guides

## Workflow

This repo uses an issue-first, PR-first workflow.

1. Open or pick an issue.
2. Create a branch.
3. Draft or edit markdown in `docs/`.
4. Open a pull request.
5. Merge to `main`.
6. GitHub Pages publishes the merged markdown.

## Branch naming

- `docs/compare-amex`
- `docs/compare-payhawk`
- `docs/playbook-ap-automation`
- `docs/sector-energy`
- `ops/issue-templates`

## Page checklist

Each page should include:
- useful title and intro
- practical structure
- at least one table, checklist or decision framework where relevant
- links back to https://numarqe.com/
- links to relevant product or comparison pages
- a short CTA at the end

## Tone

Write for:
- CFOs
- Finance Directors
- Heads of Finance
- AP leads
- procurement-adjacent finance teams

Keep tone:
- practical
- commercially aware
- specific
- low-hype

## Planning

Internal planning does not live in this public repository.

Keep only publishable content and public-safe contributor guidance here.
